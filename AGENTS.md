# Geek GIS Study — 工作区指南

## 一句话定位

本工作区是一个 WebGIS 全栈学习项目（pnpm monorepo），目标是循序渐进地学习前端全栈开发和 WebGIS 技术。

## 信息来源指引

- **学习进度**：查看 `notes/` 目录下的笔记文件，最近的笔记反映最新进度和问题记录
- **项目运行**：参照 `README.md`
- **交互规范**：见下文

## 学习协作方式

本工作区的核心目标是"学习"，不是"交付"。每次交互按以下方式协作：

### 1. 回答风格分领域

| 领域 | 回答方式 |
|------|---------|
| **前端**（Vue / Vite / JS / CSS / HTML） | 简述逻辑即可，直接给代码，无需详细讲解 |
| **后端**（Express / Node.js / 中间件） | 中等详细度，解释关键思路，必要时引导 |
| **GIS / 空间计算**（PostGIS / 坐标系 / 空间函数 / 地图原理） | **详细展开**：讲透原理和背景知识，逐步引导，必要时停下来让用户自己动手写代码 |
| **Docker / DevOps** | 简述用法，解释为什么这么做 |

### 2. 学习笔记

- 以要点、表格、对比为主，不贴大段代码
- 核心知识点保留关键代码片段（不超过 10 行）
- 笔记文件位于 `notes/` 目录

### 3. 代码生成风格

- 生成的生产代码中不写注释（除非用户明确要求）
- 代码风格遵循项目已有惯例
- 优先修改现有文件，不新建不必要的文件
- 不主动创建文档/README 文件，除非用户要求

### 4. 每次新会话快速回顾

> WebGIS 全栈学习项目（monorepo）。前端方面简述逻辑直接给代码；后端/GIS 方面要详细讲解原理并引导式学习。代码不写注释。学习进度去 `notes/` 目录下查看最新笔记。
